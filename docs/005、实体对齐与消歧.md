## 模块五：实体消歧与关系补全 设计方案

### 1. 模块定位

模块五在Causis中承接上游的“模糊地带”，负责完成两项关键的知识增强任务：**实体消歧**和**关系补全**。

- **实体消歧**：解决模块三事实层因置信度不足而遗留的模糊实体对齐问题。将指向同一物理实体的不同数据记录合并，赋予全局唯一的稳定ID。
- **关系补全**：基于已有的知识图谱结构，推断并补全实体间缺失的关系，提升知识图谱的密度和连通性。

**核心职责：**

| 职责                     | 说明                                                         |
| :----------------------- | :----------------------------------------------------------- |
| **模糊实体消歧**         | 解决模块三中置信度低于阈值的待消歧实体，包括L3级的模糊匹配实体 |
| **跨源实体链接**         | 将来自不同数据源、不同表达方式的记录正确链接到相同的全局唯一实体标识 |
| **关系补全与图谱增强**   | 基于已有结构和语义信息，预测实体间的缺失关系并补充到知识图谱 |
| **多模态实体对齐**       | 利用文本、图像、表格等多种模态信息进行更精准的实体匹配与消歧 |
| **消歧结果的可解释输出** | 为每次消歧决策提供可审计的证据和推理路径                     |

**边界划分：**

| 排除项                  | 归属                             |
| :---------------------- | :------------------------------- |
| 精确实体匹配（L1/L2级） | 模块三（基于规则的精确字段匹配） |
| 冲突的最终裁决          | 模块六（冲突协调引擎）           |
| 因果推理                | 模块七（因果推理引擎）           |
| 图谱的日常维护与更新    | 模块四（知识图谱存储层）         |
| 知识图谱的版本管理      | 模块四（通过时间感知边实现）     |

### 2. 核心设计原则

- **渐进式消歧**：采用“精确匹配 → 高置信度规则 → 模糊语义 → 交互式人工介入”的分级策略，逐级处理不同难度的消歧任务，最大化效率。
- **KG与LLM互补**：将知识图谱的结构化知识与大模型的语义理解能力进行深度融合，使两者优势互补——KG提供精确的实体关系和候选约束，LLM提供语义推理和解释能力。
- **轻量可嵌入**：消歧算法全部以Rust实现或通过FFI/ONNX加载轻量模型，最终编译为单一二进制，零外部依赖。
- **全链路可溯**：每次消歧和关系补全决策均记录完整的证据链，包括相似度分数、匹配字段、使用的模型版本和决策时间戳。

### 5. 实体消歧功能设计

#### 5.1 三级渐进式消歧流水线

```
输入: 待消歧实体 (entity_id_1, entity_id_2, 元数据, 上下文)
  │
  ├─ [Level 1] 精确属性匹配
  │   条件: 强ID字段匹配、带时间窗口的类型匹配
  │   置信度: > 0.95 → 自动合并
  │   引擎: 规则引擎 (Rust)
  │
  ├─ [Level 2] 语义相似度匹配
  │   条件: L1未决的实体对，通过语义嵌入向量计算余弦相似度
  │   置信度: 0.7~0.95 → 自动合并，附带置信度
  │   引擎: ONNX Runtime (BGE/GTE) / Candle
  │
  └─ [Level 3] LLM辅助可解释判别
      条件: L2未决的高模糊性实体对，需结合KG上下文进行深度推理
      方法: 构建包含KG子图上下文的结构化Prompt，交由大模型进行推理对齐
      置信度: < 0.7 → 提交人工审核队列
      引擎: 轻量或外部大模型（可通过API调用）
```

#### 5.2 核心算法实现策略

**L2 语义匹配**：基于预训练的句子嵌入模型，利用KG的实体层级结构和描述信息，通过逐步剪枝候选空间来提升精度。模型以ONNX格式嵌入Causis进程，由Candle（纯Rust）或ONNX Runtime直接加载推理，无需外部服务。

**L3 LLM可解释判别**：参考RADAR的分辨性推理范式，将实体对齐从生成式模式匹配重新定义为分辨性实体选择，通过强化学习增强实体的相对可分离性，使推理直接在表示空间中操作，避免生成引入的幻觉。同时借鉴EARAG框架，构建包含多源KG结构信息和实体描述的Prompt，引导大模型不仅判断实体等价性，还生成人类可理解的解释。

**不确定性校准**：当小模型（如TinyBERT级别蒸馏模型）的预测置信度低于阈值时，自动升级到更大模型或人工审核。借鉴FUSER框架的两级校准方法：属性级和实体级的不确定性估计，对候选匹配进行排序，最小化LLM幻觉引入的错误。

#### 5.3 人机协作机制

- **审核接口（REST API）**：`GET /api/v1/disambiguation/queue` 获取待审核队列，`POST /api/v1/disambiguation/{task_id}/resolve` 提交人工决策。
- **消歧溯源**：每条决策记录原始证据，包含匹配字段、分数、模型版本及时间戳。
- **主动学习**：人工决策反馈回模型训练集，周期性微调语义匹配模型。

### 6. 关系补全功能设计

#### 6.1 混合预测流水线

```
输入: 知识图谱中的实体对 (h, t)，目标：预测缺失的 r
  │
  ├─ [Stage 1] 结构评分
  │   方法: 使用 TransE / DistMult 等模型计算三元组 (h, r, t) 的结构似然分数
  │   模型: 在Causis现有KG上增量训练，模型以Candle格式存储
  │   输出: 每个候选关系的初始结构分数
  │
  ├─ [Stage 2] 语义重排
  │   方法: 利用BGE句子嵌入计算三元组文本描述的“语义连贯性”分数
  │   核心优势: 免训练，即插即用，有效捕捉语义细微差别
  │   输出: 结构分数与语义分数的加权融合结果
  │
  └─ [Stage 3] 多跳路径增强推理
      方法: 利用强化学习在图上搜索高质量的推理路径，辅助预测
      特点: 生成可解释的推理路径，而非仅仅输出预测结果
      输出: 预测的三元组 + 推理路径解释
```

#### 6.2 多模态关系补全

- **视觉+文本融合**：对于包含图片的实体（如产品图、档案照），同时利用视觉特征和文本描述进行关系预测。
- **语义自适应小样本补全**：利用LLM从实体的邻域信息中提取和生成丰富的语义描述，在小样本场景下显著提升关系补全效果。

#### 6.3 增量更新与持续学习

- 新事实经模块三进入图谱后，触发受影响实体子图的嵌入增量更新。
- 关系补全模型周期性（如每周）基于最新图谱全量重训，日常增量微调。
- 人工确认的补全结果作为正反馈信号，强化后续预测。

### 10. 总结

模块五通过**三级渐进式消歧**与**结构-语义-路径三重关系补全**的混合架构，在单一Rust进程中解决了Causis知识图谱的模糊实体消歧和缺失关系补全两大核心挑战。

- **高性能**：所有模型均以ONNX或Candle格式在进程中直接推理，零网络开销。
- **可解释性**：每次决策保留完整证据链和推理路径，确保透明度。
- **轻量化**：无需GPU、无需模型服务、无需Docker，单二进制交付。
- **可演进**：L3大模型消歧和强化学习多跳推理作为高级特性，按需引入，不影响核心管线的稳定性。
