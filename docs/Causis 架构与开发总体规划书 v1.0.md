## Causis 架构与开发总体规划书 v1.0

### 1. 项目愿景与产品定义

**Causis** 是一款以因果推理为核心、内建全链路可信追溯的AI原生知识引擎。它定位为“因果即服务（Causal as a Service）”基础设施，赋能开发者、ISV和系统集成商快速构建可解释、可审计的智能决策应用。Causis 不仅是一个数据库，而是一个从多源异构数据接入、治理、因果建模到推理审计的完整平台。

**Slogan**：每一次决策，都有可信内核。

**核心差异化**：
- 因果推理与形式化干预分析（do-operator）
- 全链路溯源至原始文件段落
- 多源冲突自动检测与四层渐进裁决
- 纯Rust轻量嵌入式部署（<100MB单二进制）
- MIT/Apache 2.0 全开源

---

### 2. 系统架构全景

Causis 采用分层架构，包含 **数据接入层、证据湖与事实层、知识图谱层、因果推理层、可信解释与审计层**，并通过统一的 API 网关对外服务。

```
┌──────────────────────────────────────────────────────────┐
│                    API 网关 (REST/GQL)                     │
├──────────────────────────────────────────────────────────┤
│  模块八  可信解释与审计引擎                                 │
│          (证据链构建、NL解释、因果图渲染、审计报告)         │
├──────────────────────────────────────────────────────────┤
│  模块七  因果推理与查询引擎                                 │
│          (DeepCausality + why-rs + CIfly)                │
├──────────────────────────────────────────────────────────┤
│  模块六  冲突检测与协调引擎                                 │
│          (四层裁决：规则→质量→LLM→人在回路)                │
├──────────────────────────────────────────────────────────┤
│  模块五  实体消歧与关系补全                                 │
│          (三级渐进式消歧 + 混合关系补全)                    │
├──────────────────────────────────────────────────────────┤
│  模块四  知识图谱层 (LadybugDB)                            │
│          (属性图存储、Cypher/SQL、向量混合检索)             │
├──────────────────────────────────────────────────────────┤
│  模块三  事实层 (Apache Iceberg + Polars/DuckDB)          │
│          (数据清洗、标准化、冲突标记)                       │
├──────────────────────────────────────────────────────────┤
│  模块二  证据湖原始层 (RustFS + Iceberg 快照)              │
│          (原始文件版本化存储)                               │
├──────────────────────────────────────────────────────────┤
│  模块一  多源数据接入与适配 (Ingestor-Core, DocTor)        │
│          (文件/数据库/云端的智能抽取与流批一体)             │
└──────────────────────────────────────────────────────────┘
```

### 3. 模块清单与分工

| 模块                          | 核心职责                                                     | 关键组件                                    |
| ----------------------------- | ------------------------------------------------------------ | ------------------------------------------- |
| **模块一** 多源数据接入       | 支持PDF/Word/Excel/PPT/关系型数据库/云端对象存储的智能解析与CDC同步 | Ingestor-Core (Rust), DocTor (Rust), Polars |
| **模块二** 证据湖原始层       | 原始文件的不可变存储与版本控制                               | RustFS (替代MinIO), Apache Iceberg 快照     |
| **模块三** 事实层             | 数据清洗、标准化、实体初步对齐（L1/L2）、冲突检测标记        | Apache Iceberg, DuckDB/DataFusion, DataProf |
| **模块四** 知识图谱层         | 属性图存储、查询、向量混合检索、因果边存储                   | LadybugDB (Cypher/SQL, HNSW)                |
| **模块五** 实体消歧与关系补全 | 模糊实体消歧（L3）、缺失关系预测                             | Candle/ONNX模型，图嵌入推理                 |
| **模块六** 冲突协调           | 四层裁决（确定性规则/质量加权/LLM辅助/人在回路），最终采信   | Rust裁决引擎，LLM API适配                   |
| **模块七** 因果推理           | 因果发现(why-rs)，干预/反事实/根因定位(DeepCausality)，图加速(CIfly) | DeepCausality, why-rs, CIfly                |
| **模块八** 可信解释与审计     | 证据链构建、自然语言解释生成、审计报告、可视化数据           | 自研解释器，审计日志(LadybugDB)             |

### 5. 分阶段实施路线图

#### 阶段一：MVP验证期（0～12个月）
**目标**：跑通请假审批自动分发场景，完成核心技术闭环，获取1-2家灯塔客户。

**交付范围**：
- 模块一：DocTor（PDF/DOCX/XLSX）+ Postgres CDC 连接器
- 模块二：RustFS + Iceberg 快照，原始文件版本化接入
- 模块三：基于Polars的数据清洗管道，L1/L2实体对齐，冲突标记SQL
- 模块四：LadybugDB 单节点部署，Cypher/SQL 查询接口，基础图谱构建
- 模块五：规则+语义向量消歧（L1/L2），基本关系补全
- 模块六：确定性规则裁决（时间优先/权威优先）与加权投票
- 模块七：DeepCausality 集成，实现 L1 因果链追溯；基本干预查询
- 模块八：证据链构建器（溯源至事实表），自然语言模板解释
- **整体产出**：单二进制部署包，请假审批场景端到端可用，决策准确率100%且推理链可读

**资源需求**：3-4名Rust工程师，1名前端（管理界面），1名产品/测试

#### 阶段二：能力泛化期（12～24个月）
**目标**：平台化，打造低代码因果工坊，赋能ISV，完成15-20家付费客户。

**新增/增强**：
- lakeFS完整集成（证据湖分支/标签）
- 模块五 L3 大模型辅助消歧上线
- 模块六 LLM辅助裁决 + 人在回路接口
- 模块七 why-rs 离线因果发现，干预/反事实全量功能
- 模块八 审计报告自动生成（PDF/HTML），交互式因果图数据
- 因果模型市场 MVP：模板发布、共享、评价机制
- 配套开发工具：Causis SDK (Python/Rust)，CLI 管理工具

**商业模式**：企业版订阅（年费）+ 市场上架分成（15%）

#### 阶段三：生态基础设施期（24～36个月）
**目标**：成为行业标准的可信决策底座，构建完善生态。

- 多集群分布式部署支持（LadybugDB 集群，元数据服务 HA）
- 完全体神经数据库操作探索（向量空间直接推理）
- 具身智能与物理世界因果推理试点
- 科学发现引擎（跨学科知识图谱对齐，因果假设生成）
- 全功能的 Causal Marketplace：应用市场、开发者认证、Token激励
- 与主流AI Agent框架（LangChain, AutoGen, CrewAI）原生集成

### 6. 关键模块接口定义

| 接口               | 方向                 | 数据格式                              | 协议                  |
| ------------------ | -------------------- | ------------------------------------- | --------------------- |
| 模块一 → 模块二    | 文件流 + 元数据      | Arrow RecordBatch                     | gRPC/file             |
| 模块二 → 模块三    | Iceberg 表路径       | Iceberg Snapshots                     | DuckDB/DataFusion SQL |
| 模块三 → 模块四    | 事实行（含溯源ID）   | Iceberg 表，LadybugDB 节点/边         | LadybugDB Cypher      |
| 模块四 ↔ 模块五    | 实体子图，消歧结果   | 图查询 JSON                           | Rust FFI              |
| 模块三/四 → 模块六 | 冲突组ID，上下文子图 | JSON                                  | 内部 service          |
| 模块六 → 模块三/四 | 裁决后事实值         | MERGE INTO Iceberg / UPDATE LadybugDB | SQL/Cypher            |
| 模块四 → 模块七    | 因果边子图           | LadybugDB 导出邻接表                  | 内存结构/FFI          |
| 模块七 → 模块八    | 推理审计ID，路径     | JSON                                  | 函数调用              |
| 模块八 → API       | 解释JSON，报告       | REST                                  | HTTP/JSON             |

### 7. 开发工具链与规范

- **版本控制**：Git (GitHub)，主干开发 + 特性分支，提交信息遵循 Conventional Commits
- **持续集成**：GitHub Actions，自动运行 `cargo test`、`cargo clippy`、`cargo fmt` 检查
- **测试策略**：
  - 单元测试覆盖核心逻辑（每个crate > 80%）
  - 集成测试覆盖关键数据流（接入→推理→解释）
  - 基准测试（因果推理延迟、证据链溯源耗时）
- **发布管理**：语义版本 (SemVer)，每个模块独立crate，定期发布二进制包与Docker镜像
- **文档**：mdBook 生成用户文档与API参考，Causal Blueprints 提供场景最佳实践

| **竞品快速跟进**             | 先发优势被填平               | 聚焦因果推理+溯源+可审计的组合壁垒；通过开源协议构建社区护城河；加速生态建设，形成网络效应 |
| **团队招聘困难**             | Rust + 因果AI复合人才稀缺    | 提供远程协作与股权激励；与高校因果AI课题组合作；内部培养与社区吸纳结合 |

### 9. 生态建设与商业模式

#### 商业模式
- **社区版 (CE)**：MIT开源，单节点，基础功能，适合开发者试用与小团队部署
- **企业版 (EE)**：商业授权，含集群、高可用、安全审计、对接 SSO/LDAP、技术支持，年费订阅（按节点数或数据量）
- **云服务 (Cloud)**：托管版，按查询次数/存储量计费，2027年启动
- **Causal Marketplace**：平台上架应用/模型交易，平台抽佣 15%-20%

#### 生态运营
*   **官方因果模板与工具包**：发布多个高价值行业因果骨架、预训练模型、SDK，降低开发门槛。
*   **Causis Benchmark与社区贡献者激励**：设立公开的因果推理性能榜单和奖励计划（如算力券），鼓励开发者贡献模型和知识产权，促进良性竞争。
*   **灯塔客户联合开发**：与1-2家金融/医疗客户深度合作，打造行业标杆案例，将场景最佳实践沉淀为“Causal Blueprints”。
*   **开发者关系**：举办黑客松、因果AI论文解读、全球巡回Meetup，建立社区影响力。
