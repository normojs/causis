# Causis

[English README](README.md)

Causis 是一个 Rust 优先的因果知识引擎，面向可解释、可审计的企业决策。

它的定位是 **因果即服务（Causal as a Service）** 基础设施：为开发者、ISV 和系统集成商提供可嵌入的决策底座，用形式化因果推理、多源数据治理和全链路证据溯源来构建可信决策应用。

> 每一次决策，都有可信内核。

## 当前状态

Causis 当前处于架构与规划阶段。本仓库主要包含产品战略、模块设计、竞品分析和实施规划文档。运行时代码和工程脚手架尚未加入。

## Causis 是什么

Causis 不只是数据库、GraphRAG 工具或垂直行业应用。它希望组合以下能力：

- **因果推理**：因果链追溯、干预分析、反事实推演和根因查询。
- **证据溯源**：每条事实和决策都能追溯到原始文件、位置、版本和处理步骤。
- **冲突协调**：多源矛盾能够被发现、保留、裁决和审计。
- **知识图谱基础设施**：时态属性图、规则建模、图遍历和图向量混合检索。
- **私有化部署**：面向嵌入式和私有环境的轻量 Rust 架构。

## 架构

Causis 分为八个核心模块：

```text
API 层
  REST / GraphQL / SDK / CLI

可信解释与审计
  证据链、报告、图可视化、审计日志

因果推理与查询引擎
  因果追溯、干预、反事实、根因定位

冲突协调
  规则裁决、质量加权、LLM 辅助、人在回路

实体消歧与关系补全
  模糊实体匹配、语义匹配、关系预测

知识图谱
  时态属性图、Cypher/SQL、图向量检索

事实层
  清洗、标准化、质量评分、冲突标记

证据湖
  不可变原始证据、元数据、溯源、版本快照

多源数据接入
  文档、数据库、对象存储、流数据
```

## 模块概览

| 模块 | 职责 | 当前设计方向 |
| --- | --- | --- |
| 多源数据接入 | 解析文档并连接结构化数据源 | Ingestor-Core、DocTor、Arrow 流 |
| 证据湖 | 保存不可变原始证据与元数据 | RustFS、Apache Iceberg、溯源表 |
| 事实层 | 标准化事实并标记冲突 | Polars、DuckDB/DataFusion、质量评分 |
| 知识图谱 | 构建时态图知识 | LadybugDB、Cypher/SQL、向量检索 |
| 实体消歧 | 处理模糊实体和关系补全 | 规则、嵌入、ONNX/Candle、审核 API |
| 冲突协调 | 透明裁决可信事实版本 | 四层渐进式裁决流程 |
| 因果推理 | 执行形式化因果查询 | DeepCausality、why-rs、CIfly |
| 解释与审计 | 生成带证据链的解释 | 证据链构建器、报告、审计日志 |

## MVP 方向

建议第一个纵向切片采用“请假审批自动分发”场景：

1. 接入制度文件、员工数据和审批数据。
2. 保存带稳定锚点的原始证据。
3. 标准化事实并标记冲突。
4. 构建一个小型时态知识图谱。
5. 必要时裁决审批链冲突。
6. 执行因果链追溯查询。
7. 返回包含证据链接和审计记录的解释。

这个路径可以聚焦验证 Causis 的核心价值：决策可计算、可解释、可追溯。

## 路线图

当前规划文档将项目分为三个阶段：

1. **MVP 验证**：完成一个端到端决策场景，验证带证据链的因果追溯。
2. **平台化**：泛化 API、SDK、低代码工具和人工审核流程。
3. **生态化**：建设因果模型市场、合作伙伴集成和行业模板。
